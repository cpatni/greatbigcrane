Great Big Crane allows managing python buildouts and django projects. Unlike many django applications, it is designed to be run on a local secured environment. This allows it to be extremely responsive and dispenses with user authentication. It also allows us to do cool things like open your text editor.

Great Big Crane requires a recent browser, as we're using HTML 5, CSS 3 and a pile of vendor-specific properties.

In normal usage, Great Big Crane allows you to manage a collection of projects in a specific code directory. This directory name is set on the preferences page. For example, I store my code in /home/dusty/code/ and individual projects are in /home/dusty/code/greatbigcrane/ or /home/dusty/code/whohasmy/. The projects are expected to contain a standard buildout structure with a buildout.cfg and bootstrap.py. Great Big Crane can introspect existing directories and buildouts in this format or it can help set up brand new buildouts.

To see it in action, click the Projects button on the menu bar and then click the Add Project button to add a new project. This simple form allows you to enter the name, base directory, an optional git repository and description. The base directory is automatically populated from the directory preference and the name of the project, however you are free to change this for code in non-standard locations.

If you provided a git repository, it will automatically clone the repository into the new working directory. Otherwise, it will initialize and boostrap a simple skeleton buildout in that directory. Both the clone and bootstrap actions are processed in an external job queue, and notifications will show up on your dashboard when they have completed.

The dashboard lists your most recently active projects and explicitly favourited projects. Projects can be favourited using the star icon in the project list. You can also schedule a buildout and tests from the dashboard. Tests are run through the buildout. The test icon will glow either red or green depending on whether or not the last set of tests for that project completed successfully or not.

Notifications indicate the output of various asynchronous commands, and come in three flavours: success, failure, or general. The small "x" box on the right of a notification allows you to dismiss it. The notifications page allows you to review all notifications dismissed or not at the same time. Clicking a notification allows you to view the output of the command that was run.

If you click the project's row, you can see the buildout configuration for that project, with some helpers to edit the buildout. This information is introspected from the buildout.cfg itself, so if you want to edit it directly in your text editor, you can. We have provided a nifty line editor for editing key value pairs in sections such as eggs or extra paths that often have multiple lines. Certain expansions are available for quickly referring to other sections (e.g. eggs = ${buildout:eggs}). Select an expansion from the dropdown and click the insert button to apply it.

The sidebar on project lists displays non-dismissed notifications for the project, as well as a set of actions that can be executed on that project, mostly asynchronously through the job queue:

  • edit a project's details
  • delete a project it
  • run buildout on the project
  • bootstrap the buildout
  • run the test suite
  • if the project has a git repo, you can issue a git pull to update it
  • if it is a github project, you can view the github page to review commits and issues
  • if it is a django project, you can run syncdb, startapp, and migrate
  • if you've configured the Editor Path (and possibly the Terminal Path), you can edit the buildout.cfg in your favourite editor
