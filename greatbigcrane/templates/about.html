Great big crane allows managing python buildouts and django projects. Unlike many django applications, it is designed to be run on a local secured environment. This allows it to be extremely responsive and dispenses with user authentication.

In normal usage, great big crane allows you to manage a collection of projects in a specific code directory. This directory name is set in the preferences page. For example, I store my code in /home/dusty/code/ and individual projects are in /home/dusty/code/greatbigcrane/ or /home/dusty/code/whohasmy/. The projects are expected to contain a standard buildout structure with a buildout.cfg and bootstrap.py. Great big crane can introspect directories and buildouts in this format, or it can help set up brand new buildouts.

To see it in action, hit the Projects button on the menu bar and then click the Add Project button to add a new project. This simple form allows you can enter the name, base directory, an optional git repository and description. The base directory is automatically populated from the directory preference and the name of the project, however you are free to change this for code in non-standard locations.

If you provided a git repository, it will automatically clone the repository into the new working directory. Otherwise, it will initialize and boostrap a simple skeleton buildout in that directory. Both the clone and bootstrap processes are processed in an external job queue, and notifications will show up on your dashboard when they have completed.

The dashboard lists your most recently active projects and explicitly favourited projects. Projects can be favourited using the star icon in the project list. You can also schedule a buildout and tests from the dashboard. Tests are run through the buildout. The test icon will glow either red or green depending on whether or not the last set of tests for that project completed successfully or not.

Notifications indicate the output of various asynchronous commands, and can indicate success, failure, or a general notification. The small "x" box on the right of a notification allows you to dismiss it. The notifications page allows you to visually review all recent notifications and is set up so you can spatially see notifications in both chronological order and sorted by status (success, general, failure) at the same time. Clicking the notification allows you to view the output of the command that was run.

If you click the project's option, you can see the buildout configuration for that project, with some helpers to edit the buildout. This information is introspected from the buildout.cfg itself, so if you want to edit it directly in your text editor, you can. We have provided a nifty line editor for editing key value pairs in sections such as eggs or extra paths that often have multiple lines.

The sidebar on project lists not only non-dismissed notifications for the project, but also a set of actions that can be executed on that project, mostly asynchronously through the job queue. You can edit the project details or delete it. You can run buildout on the project or bootstrap the buildout. You can run the test suite. If the project has a git repo, you can issue a git pull to update it. If it is a github project, you can view the github page to review commits and issue. If it is a django project, you can run syncdb and migrate.
